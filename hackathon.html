Drone Defense Hackathon

Event Overview

In November 2025, I had the privilege of being selected as one of the 200 finalists from a pool of over 800 applicants to compete in the AdoptAI Hackathon over a slightly less than 3 days period. This prestigious event hosted the Armed Forces for a specific challenge titled "Delivery in Crisis".

The goal was to design a robust, autonomous logistics system capable of operating in high-stress environments. Our team secured 3rd place by engineering a comprehensive solution that addressed the entire supply chain, from the soldier on the field to the autonomous drone fleet.

The Challenge: Delivery in Crisis

The core of the challenge was to manage a fleet of drones to deliver supplies (medical kits, ammunition, rations) to various units deployed on the field. The environment was dynamic, requiring real-time adaptation to changing threats and urgent requests. We had to ensure:

  • Fast response times for critical requests.
  • Efficient mission allocation to available drones.
  • Safe trajectory planning to avoid obstacles and no-fly zones.

Our Solution Architecture

To solve this complex problem, we designed a three-layered architecture ensuring seamless communication between human operators and autonomous machines.

1. Field Unit Interface

The first layer is the "client" side. We designed a simplified interface for units on the ground. In a crisis situation, soldiers cannot type complex commands. Our GUI allows them to:

  • Request specific supplies with a single tap.
  • Broadcast their GPS position automatically.
  • Set the priority level of the request (Standard vs. Critical).

Soldier's Interface Demonstration

To go into technical detail, soldiers log into the application prior to deployment to authenticate their identity, unit, and rank. Once in the field, when a soldier needs to order supplies, the interface prioritizes items based on their specific role, while still allowing access to the full catalog if necessary.

Once the supplies are confirmed, the user proceeds to the planning screen. Here, they can set the order priority, choose between immediate delivery or a scheduled time, and finally select the delivery location. This can be done by tapping a position on the tactical map or by manually entering coordinates in their preferred format (MGRS, DMS, DD, etc.).

Additionally, to mitigate security risks—such as an enemy acquiring a lost device—we implemented a multi-factor authentication system. Each squad is assigned a rolling security code (changing daily, weekly, or monthly) required to confirm orders and access the unit's request history.

2. Logistics Command Dashboard

The second layer is the Supervisory Dashboard designed for the logistics operators. This central hub provides a global view of the theater of operations. It allows operators to:

  • Visualize all active units and drone positions in real-time.
  • Approve or deny incoming supply requests.
  • Monitor drone battery levels and status.
  • Override autonomous decisions if tactical intervention is needed.

Logistics Command Dashboard Demonstration

The dashboard was built with a focus on clarity and rapid decision-making. Operators can filter requests by priority, view drone statuses at a glance, and receive alerts for any anomalies (e.g., low battery, communication loss). Crucially, the dashboard is designed to reduce the operator's cognitive load, enabling them to focus on critical decisions. The backend algorithms handle the majority of routine tasks and logistical calculations.

The right panel displays all pending missions with their priority levels and coordinates. By pressing the "auto-assign" button, the backend algorithms will efficiently allocate missions to available drones. If necessary, the operator can also drag and drop missions to specific drones or time slots; the system provides immediate warnings if a selected drone is unsuitable for the task. Finally on the left pannel, the operator can still have a complete overview of the order history.

3. Algorithms & Real-Time Trajectory

The "brain" of our solution lies in the backend algorithms. We implemented advanced logic to handle the autonomy:

  • Mission Allocation: An optimization algorithm that assigns the nearest available drone with sufficient battery and compatible specs to the highest priority request.
  • Real-Time Trajectory Planning: A path-planning engine that calculates the safest route, dynamically updating to avoid static obstacles and dynamic threats defined by the command center.

We incorporated numerous parameters to ensure optimal algorithm performance. The system manages three distinct drone models, each with specific characteristics (payload, speed, range). The algorithm evaluates these constraints alongside environmental factors—such as weather conditions, terrain type (urban, forest, desert), travel distance, mission priority, and risk levels—to prevent deploying high-value assets on high-risk missions.

The simulation also accounts for hostile environments, including GNSS-denied zones, which the trajectory planning engine actively avoids. Strict No-Fly Zones (NFZ) were implemented; if a drone ceases telemetry transmission (indicating loss or destruction), the system automatically generates a dynamic NFZ around its last known coordinates to protect the remaining fleet.

Finally, the solution features multi-role versatility. If a drone determines it cannot complete its return trip due to damage or critical battery levels, it can switch to an offensive loitering munition mode to target enemy units rather than being lost in transit.

Results

Our solution was successfully demonstrated during the final presentation, showcasing a fully functional loop from a soldier's request to the drone's autonomous takeoff and pathfinding. The robustness of our mission allocation algorithms and the clarity of our supervisory dashboard were key factors in awarding us the 3rd place podium finish.
